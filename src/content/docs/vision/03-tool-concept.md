## コンセプト

「プロセステーラリング(組織標準プロセスをプロジェクト文脈に合わせて仕立て直す作業)」を、
体系化された知識ベースの上でインタラクティブに行えるツール。

## 想定する入出力

### 入力(チーム・プロダクトのコンテキスト)

- **チーム体制**: アサイン可能なメンバー、スキルセット、稼働時間、専任/兼任
- **組織構造**: 品質管理部門・セキュリティ部門・CTO レビューなどの有無、決裁ゲートの要件
- **プロダクト特性**: 期待品質、ドメインの複雑さ、規制要件
- **事業フェーズ**: PoC / MVP / グロース / 安定運用
- **AI 利用条件**: 利用可能なモデル・ツール、コンピューティングリソース、セキュリティ制約

### 出力(最適プロセスの提案)

- 推奨する開発プロセスの全体像(フェーズ・ロール・ゲート・成果物)
- 各ロールにおける生成AIの使い方(置き換え / 支援 / 人によるチェックポイント)
- プロセス運用の実装方法(Git戦略、CI/CD ゲート構成、AI 実行環境)
- 採択までの流れ: 提案 → チームで修正 → 採択 → 運用方法の策定

## 実装方針(暫定)

- 本サイト(Astro + Starlight)上に、Svelte/React コンポーネントをアイランドとして埋め込む形で実装
- 知識ベース(プロセス定義)はドキュメントと同じリポジトリで docs-as-code 管理し、ツールはそれを参照する
