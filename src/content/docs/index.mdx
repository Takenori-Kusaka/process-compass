## このプロジェクトがやること

<CardGrid stagger>
  <Card title="既存プロセスの体系化" icon="open-book">
    ウォーターフォール、アジャイル/スクラム、TDD、DDD、イベント駆動、仕様駆動(SDD)——
    ロールモデル・決裁ゲート・レビュープロセス・成果物まで踏み込んで、現行の開発プロセスを体系的に整理します。
  </Card>
  <Card title="AIDLC の現実的な導入設計" icon="rocket">
    「プロダクトオーナーだけのチームで自律開発」という理想論と、
    組織的な責任・品質保証の現実とのギャップを分析し、実際に導入できるプロセスへ落とし込みます。
  </Card>
  <Card title="図解ファースト" icon="puzzle">
    新しい概念を未経験の人にも理解してもらうため、シーケンス図・ロール図・階層構造図など、
    Mermaid / UML / 生成画像を駆使した図解中心のドキュメントを作ります。
  </Card>
  <Card title="プロセス提案ツールへ" icon="setting">
    最終的には、アサイン可能なメンバー・稼働時間・体制・期待品質・事業フェーズを入力すると、
    最適な開発プロセスとその運用方法を提案するインタラクティブなツールとして確立します。
  </Card>
</CardGrid>
