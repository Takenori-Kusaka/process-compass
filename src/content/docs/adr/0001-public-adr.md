## ステータス

採用(2026-07-09)

## コンテキスト

表現ポリシーや技術選定など、プロジェクトの方向を決める判断が増えてきた。決定の「なぜ」が残らないと、あとから参加する人(将来の自分を含む)が経緯を追えず、同じ議論を繰り返すことになる。本プロジェクトは開発プロセスの検討自体がテーマであり、意思決定の残し方も成果物の一部といえる。

## 決定

重要な意思決定を ADR として `src/content/docs/adr/` に記録し、GitHub Pages 上で公開する。運用ルールは[決定記録(ADR)とは](/process-compass/adr/)に定める。

## 検討した選択肢

- **リポジトリ内の非公開ディレクトリ(docs/adr/)に置く** — GitHub を開かないと閲覧できず、サイトの検索対象にも載らないため見送り
- **Issue のみで記録する** — 決定と議論が混ざり、現行有効な決定の一覧性がないため見送り

## 影響

- 決定のたびに ADR を書くコストが発生する(1件30分程度を想定)
- 決定の変更は新 ADR による置き換えとして履歴が残る
- 公開されるため、外部からのフィードバック対象になる(本プロジェクトの方針に合致)
