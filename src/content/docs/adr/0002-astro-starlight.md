## ステータス

採用(2026-07-08)

## コンテキスト

ドキュメント主体のプロジェクトだが、最終ゴールはチーム体制などを入力すると最適プロセスを提案するインタラクティブなツールであり、将来 UI コンポーネントを組み込める基盤が必要だった。日本語主体で始めて、あとから英語版を追加できる構造も求められた。

## 決定

Astro + Starlight を採用する。日本語をルートロケールに設定し、将来 `locales.en` を追加するだけで多言語化できる構造を維持する。

## 検討した選択肢

- **SvelteKit** — ツール化の自由度は最大だが、検索・目次・多言語などドキュメント機能を自作する初期コストが大きく見送り
- **VitePress** — 最速で立つが、インタラクティブなツール組み込みの自由度が低く見送り
- **Docusaurus** — i18n は強力だがビルドが重く、React 固定になるため見送り

## 影響

- ドキュメント機能(検索・サイドバー・多言語)は Starlight 標準で得られる
- 将来のツール UI は Astro アイランドとして Svelte / React を選択できる(フレームワーク未確定のまま進められる)
- Starlight のメジャー更新は破壊的変更を含むことがあり、追従コストを見込む(実績: v0.39 の sidebar 形式変更)
