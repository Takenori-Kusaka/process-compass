# 作図規約の検討メモ(2026-07-08)

Issue: #17 / 清書済み → `src/content/docs/community/style-guide-diagrams.md`

## 検討の背景

- 本プロジェクトは「図解ファースト」。未経験者に新規概念(開発プロセス)を伝えるには、文章より図が効く
- 図の種類が書き手ごとにバラバラだと読者の学習コストが上がるため、記法選択とスタイルを規約化する

## 手段の整理

| 手段 | 得意 | 不得意 | 出典 |
| --- | --- | --- | --- |
| Mermaid | フロー・シーケンス・状態・ガント。差分管理可能、テーマ自動追従 | 自由レイアウト、意匠性 | https://mermaid.js.org/intro/ |
| 生成画像(Gemini) | ヒーロー・概念イメージ・雰囲気の伝達 | 正確な構造表現(文字が崩れる) | scripts/generate-image.mjs |
| UML(PlantUML 等) | 厳密なクラス図・コンポーネント図 | ビルド統合が別途必要(現状未導入) | 将来必要になったら検討 |

## 決定事項(規約の骨子)

1. 構造・流れ・関係の説明は Mermaid を第一選択とする(テキスト差分でレビューできるため)
2. 生成画像は「雰囲気・比喩・ビジュアルアイデンティティ」用途に限定し、文字を入れない
3. プロセスの階層(全体 → ワークフロー → 作業)は「1ページ1階層 + サブグラフ + ページ分割」で表現する
4. 記法の選択表・スタイル規約は公開ページに掲載し、Claude Code 用には .claude/skills/mermaid-diagram に同内容の手順版を置く(二重管理を避けるため、規約変更時は両方更新する)

## 未決事項

- PlantUML / D2 などの追加 DSL → 必要になった時点で Issue 起票
