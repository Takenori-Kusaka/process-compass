# 日本語校正ワークフロー

## 手順

1. `npm run lint:text` を実行する(単一ファイルなら `npx textlint --format compact "<path>"`)
2. 検出された違反を下の対処表に従って修正する
3. 再実行してクリーンになるまで繰り返す(`--fix` は使わない。文意が壊れることがあるため手動で直す)

## ルール別の対処表

| ルール | 意味 | 直し方 |
| --- | --- | --- |
| no-mix-dearu-desumasu | 文体混在 | 本文は「ですます調」、箇条書き・表は「である調」か体言止めに統一 |
| ja-no-mixed-period | 文末に句点がない | 段落文は「。」で終える。体言止めが不自然なら「〜します。」へ |
| max-kanji-continuous-len | 漢字7連続以上 | 「の」を挟む・語を言い換える(例: 社内専門技術者集団 → 社内の技術専門家チーム) |
| no-doubled-joshi | 同一助詞の連続 | 助詞を変える・語順を入れ替える・文を分割(括弧内も同一文として数えられる点に注意) |
| ja-no-redundant-expression | 冗長表現 | 「することができます」→「できます」、「することも可能」→「〜も可能」 |
| sentence-length | 一文150字超 | 文を分割する |
| no-exclamation-question-mark | (無効化済み) | — |

## 本プロジェクト固有の注意

- 見出し・表のセル・リンクテキストは句点不要
- Mermaid コードブロック内・frontmatter は textlint 対象外なので自由に書いてよい
- ルール自体が不合理な場合のみ `.textlintrc.json` の調整を提案する(勝手に緩めない)
