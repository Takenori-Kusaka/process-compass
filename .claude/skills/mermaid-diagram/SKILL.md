# Mermaid 作図規約

本プロジェクトは「文章より図」が原則。新規概念のページでは、まず図を設計してから本文を書く。

## 記法の選択ガイド

| 説明したいもの | 記法 | 例 |
| --- | --- | --- |
| プロセスの流れ・フェーズ遷移 | `graph LR`(横)/ `graph TD`(縦) | 開発フェーズの全体像 |
| ロール間のやりとり・承認フロー | `sequenceDiagram` | DR での承認・レビュー依頼 |
| 状態遷移(チケット、成果物の状態) | `stateDiagram-v2` | Issue のライフサイクル |
| 概念の包含・分類 | `graph TD` + `subgraph` | プロセス体系の分類 |
| 時系列計画 | `gantt` | マイルストーン計画 |
| 判断分岐 | `flowchart` + 菱形ノード | テーラリング判断 |

## 階層構造の表現(本プロジェクトの核)

プロセスは「全体 → フェーズ内ワークフロー → 個別作業」の3階層で図解する。

1. 全体図は1ページに1つ。ノードは7個前後まで(それ以上は分割)
2. 掘り下げは `subgraph` か、別ページへ分けて全体図のノードから内部リンクで誘導する
3. 同じ対象を別の粒度で描くときは、上位図のノード名と下位図のタイトルを一致させる

## スタイル規約

- ノードラベルは日本語。矢印ラベルで「何が渡るか」(成果物・承認)を書く
- 流れ図は原則 `LR`(横)。階層・分類は `TD`(縦)
- 色の直接指定はしない(astro-mermaid がライト/ダークテーマに自動追従するため)
- ロールは `[人] PO`、AI は `[AI] レビュー支援` のように接頭辞で区別する

## 書き方

````markdown
```mermaid
graph LR
  A[調査] -->|調査メモ| B[体系化]
  B -->|ドキュメント| C[公開]
```
````

そのまま ```mermaid ブロックで書けば astro-mermaid がビルドに組み込みレンダリングする。プレビューは `npm run dev` で確認する。
